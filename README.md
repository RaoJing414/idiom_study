# 成语学习助手 (Idiom Study Assistant) `v0.0.2`

一个基于 React + TypeScript 构建的高颜值、功能完备的成语学习应用。本项目采用现代化的前端架构，通过 PWA、IndexedDB 和 GitHub Pages 技术，完美模拟了一个真实的生产环境应用。

---

## 📖 项目功能说明书

### 1. 核心定位

本项目旨在利用纯前端技术栈，为用户提供一个无需联网、响应极速、且具备完整数据库持久化能力的成语学习工具。它特别适合备考学习、碎片化知识巩固以及作为“全离线 Web 应用”的技术范本。

### 2. 功能模块详解

#### 🏠 智能首页 (Dashboard)

- **学习统计**：实时显示“已掌握”、“学习中”及“错题集”的数量。
- **动态计划**：根据成语的使用频率（重点、常见、一般）自动对词库进行分组，用户可以按“天”或“组”制定学习计划。
- **自测入口**：快速开启“看成语选释义”或“看释义选成语”测试。

#### 📖 浸入式学习卡片 (Study System)

- **交互设计**：采用毛玻璃特效结合卡片翻转/滑入动画，减少视觉疲劳。
- **沉浸模式**：默认隐藏释义，鼓励用户先思考，点击后通过细腻动画展示释义。
- **自动播放**：支持自动显示释义模式，适合快速扫视复习。
- **进度标记**：支持手动标记“已掌握”，标记后的成语将从学习序列中淡出。

#### 📝 双模自测系统 (Quiz Hub)

- **成语辨析测试**：根据题目随机生成四个干扰项，考查用户对成语字面与深层含义的匹配能力。
- **释义匹配测试**：反向考查用户对成语词汇量的储备。
- **实时反馈**：答题后立即显示对错，并记录到底层数据库。

#### ❌ 动态错题集 (Wrong Collection)

- **自动抓取**：所有在测试中答错的成语会自动进入错题集。
- **针对性复习**：错题集支持专项的“成语/释义”测试。
- **智能移出**：当用户在测试中重新答对该成语时，系统会自动将其从错题集逻辑中移除。

## � 部署与运行指南

本项目支持多种部署方式，既可以托管在云端，也支持在局域网内作为私有服务运行。

### 方式一：GitHub Pages 线上部署（模拟真实服务器）

利用 GitHub Pages 的免费托管能力，可以将项目作为公开网页发布。

1. **构建并发布**：
   ```bash
   pnpm run deploy
   ```
2. **说明**：该脚本会自动执行 `build`，并将生产环境代码推送到 `gh-pages` 分支。访问地址通常为 `https://[你的用户名].github.io/idiom_study/`。

### 方式二：局域网本地部署（手机与电脑同步访问）

如果你想在宿舍、办公室等局域网环境下，让手机也能访问电脑上运行的应用，请执行：

1. **启动 host 模式**：
   ```bash
   pnpm run dev --host
   ```
2. **连接步骤**：
   - 终端会输出两个地址：`Local` (本机访问) 和 `Network` (局域网访问)。
   - 确保手机和电脑连接在同一个 Wi-Fi。
   - 在手机浏览器输入 `Network` 显示的 IP 地址（例如 `http://192.168.1.10:5173/`）即可访问。
   - **体验优化**：在手机浏览器中选择“添加到主屏幕”，应用将以 PWA 模式运行，拥有接近原生 App 的全屏体验。

### 方式三：生产环境模拟调试

在本地预览打包后的效果：

```bash
pnpm run build
pnpm run preview --host
```

## 🏗️ 架构解析

- **数据层 (DB Layer)**：基于 **IndexedDB**。通过 Service Worker 和本地数据库的双重加持，即使在“飞行模式”下，用户的错题集、学习进度也永不丢失。
- **性能层 (Performance)**：所有资源（成语库、图标、脚本）在首次加载后即被强缓存。PWA (Progressive Web App) 技术确保了应用在弱网甚至断网环境下的稳定性。
- **样式层 (Styling)**：使用 Vanilla CSS 配合 CSS Variables 打造的深色/动态主题系，支持所有主流移动端浏览器。
